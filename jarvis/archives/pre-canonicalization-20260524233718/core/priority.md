# Prioritization Rules

## Priority Rules
1. URGENT: Revenue at risk, deadlines today, blockers on critical path
2. HIGH: Due within 24h, blocked tasks, important vendor/client comms
3. MEDIUM: Due within 72h, follow-ups, quotes/reviews needed
4. LOW: Informational, planning, non-urgent admin
5. SCHEDULED: Date-specific, not time-sensitive

## Escalation Path
1. Delegatee misses deadline → escalate to their manager
2. Deadline 24h past → escalate to Will (COO)
3. Revenue impact >$1000 → immediate Will alert
4. Client escalation (Globo, Record, etc.) → immediate priority
5. Inventory/logistics → same-day resolution required

## File Priority
- SOUL.md, AGENTS.md → NEVER modify
- TASKS.md, PROJECTS.md → always up-to-date before operations
- MEMORY.md → append-only, never overwrite
- HEARTBEAT.md → auto-update on scan
